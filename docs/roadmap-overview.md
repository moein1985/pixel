# پیکسل — شناسنامه هوشمند کشاورزی

## مرور کلی پروژه

پروژه «پیکسل» یک **پلتفرم ملی B2B/B2G** برای تحول دیجیتال در حوزه کشاورزی ایران است. این پلتفرم با ایجاد یک زیرساخت یکپارچه و هوشمند، کشاورزان، تأمین‌کنندگان نهاده‌ها، صنایع فرآوری کشاورزی و سازمان‌های مرتبط را به هم متصل می‌کند.

### مأموریت پروژه

- ایجاد بانک جامع اطلاعات کشاورزی کشور
- هوشمندسازی زنجیره تأمین کشاورزی
- کاهش هزینه‌های تأمین و خرید تا ۳۵٪
- تبدیل داده‌های خام کشاورزی به اطلاعات کاربردی برای تصمیم‌گیری لحظه‌ای

### مفاهیم جدید اختصاصی کشاورزی

| مفهوم | توضیح |
|---|---|
| فصل کشت و برداشت | هر محصول کشاورزی فصل کاشت و برداشت مشخصی دارد که روی قیمت و عرضه تأثیر مستقیم دارد |
| نوع کشت | دیمی / آبی / گلخانه‌ای / باغی |
| نهاده‌های کشاورزی | بذر، کود، سموم، آب، انرژی — ورودی‌های فرآیند تولید کشاورزی |
| گواهی ارگانیک / GAP | استانداردهای کیفیت کشاورزی که در پروفایل و رتبه‌بندی مؤثرند |
| زنجیره سرد | زیرساخت لجستیک برای محصولات فاسدشدنی (میوه، سبزی، لبنیات) |
| سامانه کشت | اطلاعاتی درباره نوع محصول، مساحت، موقعیت مزرعه، تاریخ کاشت و برداشت |

## Tech Stack

### Frontend

| تکنولوژی | کاربرد |
|---|---|
| Next.js (App Router) | فریم‌ورک اصلی فرانت‌اند — SSR/SSG برای سئوی پورتال |
| TailwindCSS | استایلینگ |
| shadcn/ui | کامپوننت‌های UI |
| Vazirmatn | فونت فارسی |
| dayjs + jalaliday | تاریخ جلالی |
| Zustand | state management سبک |
| React Hook Form + Zod | فرم‌ها و اعتبارسنجی |

### Backend (Main API)

| تکنولوژی | کاربرد |
|---|---|
| Node.js + TypeScript | بک‌اند اصلی |
| tRPC | API داخلی type-safe |
| tRPC-openapi plugin | تولید endpointهای REST برای اتصال به سامانه‌های خارجی |
| Drizzle ORM | ORM سبک TypeScript-native |
| BullMQ | صف کارهای async (SMS، پردازش تصویر، نوتیفیکیشن) |
| Auth.js (NextAuth) | احراز هویت با OTP پیامکی + نقش‌های متعدد |
| Socket.io | real-time chat و نوتیفیکیشن |

### AI Services (مستقل)

| تکنولوژی | کاربرد |
|---|---|
| Python + FastAPI | سرویس AI مستقل |
| Transformers (HuggingFace) | NLP فارسی و چت‌بات |
| OpenCV / Pillow | پردازش تصویر |
| scikit-learn / XGBoost | پیش‌بینی قیمت و تحلیل |
| ارتباط با Node.js | از طریق HTTP REST |

### Data Layer

| تکنولوژی | کاربرد |
|---|---|
| PostgreSQL 16 | دیتابیس اصلی |
| TimescaleDB (extension) | داده‌های زمانی (قیمت‌ها، تراکنش‌ها) |
| pg_trgm + unaccent | جستجوی فارسی fuzzy |
| Redis | cache + queue + real-time + session |
| MinIO | ذخیره فایل و تصویر (S3-compatible) |

### Infrastructure

| تکنولوژی | کاربرد |
|---|---|
| Docker + Docker Compose | containerization کل سیستم |
| Nginx | reverse proxy + SSL + static serve |
| Turborepo + pnpm | monorepo management |
| Vitest | unit و integration test (JS/TS) |
| Playwright | E2E test |
| pytest | test سرویس AI (Python) |

## استراتژی تست

هر فاز شامل سه لایه تست است:

### ۱. تست‌های خودکار (Automated Tests)
- **Unit Test** (Vitest / pytest): تست توابع و کامپوننت‌های مجزا
- **Integration Test**: تست تعامل بین ماژول‌ها و دیتابیس
- **E2E Test** (Playwright): تست سناریوهای کامل کاربر از ابتدا تا انتها
- **API Test**: تست endpointهای tRPC و REST

### ۲. تست‌های میدانی (Field Tests)
- در پایان هر فاز، قابلیت‌ها با **کاربران واقعی** (کشاورزان، تأمین‌کنندگان، ادمین) تست می‌شوند
- بازخورد کاربران جمع‌آوری و قبل از ورود به فاز بعد رفع اشکال می‌شود

### ۳. تست‌های استقرار (Deployment Tests)
- بررسی health check تمام سرویس‌ها بعد از deploy
- تست backup و restore دیتابیس
- تست rollback در صورت مشکل
